## How the changelog works

- **New hooks**: Whenever a new hook is added to useKit, it will be listed here and tagged as **New** in the Docs sidebar.
- **Improvements**: Significant API or documentation improvements to existing hooks will be noted here.
- **Breaking changes**: Any changes that require code updates in your apps will be clearly called out in future entries.

You can always browse the full hooks collection at `/docs/hooks` and use this page to see what’s changed between visits.
