title: uselab
description: A command-line tool for installing UseKit hooks into your React projects. Simplify your hook installation process with a simple command.
---

## Installation

```bash
npm install -g uselab
```

Or run directly without installation:

```bash
npx uselab@latest --help
```

## Quick Start

1. **Initialize your project:**

   ```bash
   npx uselab@latest init
   ```

2. **Install a hook:**

   ```bash
   npx uselab@latest add use-network-status
   ```

3. **View available hooks:**

   ```bash
   npx uselab@latest list
   ```

## Commands

### `add` - Install Hooks

Install one or more hooks from the registry, URL, or local file path.

#### Syntax

```bash
npx uselab@latest add [options] [hooks...]
```

#### Arguments

- `[hooks...]` - Hook name(s), URL(s), or local path(s) (space-separated)
  - **Registry names:** Accepts kebab-case or camelCase format
    - Examples: `use-network-status`, `useToggle`, `useCounter`

  - **URLs:** Full URL to a hook JSON file
    - Examples: `https://example.com/hooks/use-custom.json`

  - **Local paths:** Relative or absolute path to a local hook JSON file
    - Examples: `./hooks/use-custom.json`, `../shared/use-utils.json`

#### Options

| Option              | Description                                                       | Default           |
| ------------------- | ----------------------------------------------------------------- | ----------------- |
| `-y, --yes`         | Skip confirmation prompts. Implies `--overwrite` when files exist | `false`           |
| `--overwrite`       | Overwrite existing hook files if they already exist               | `false`           |
| `-c, --cwd <path>`  | Working directory for resolving package.json and hooks directory  | Current directory |
| `-p, --path <path>` | Custom installation path (overrides hooks config)                 | Uses config       |

#### Examples

**Install a single hook:**

```bash
npx uselab@latest add use-network-status
```

```bash
npx uselab@latest add useToggle
```

```bash
npx uselab@latest add use-counter
```

**Install multiple hooks:**

```bash
npx uselab@latest add use-counter use-toggle use-network-status
```

```bash
npx uselab@latest add useBattery useCounter useToggle
```

**Skip confirmation prompts:**

```bash
npx uselab@latest add use-counter --yes
```

```bash
npx uselab@latest add use-toggle -y
```

**Overwrite existing files:**

```bash
npx uselab@latest add use-network-status --overwrite
```

```bash
npx uselab@latest add use-counter --overwrite --yes
```

**Specify working directory:**

```bash
npx uselab@latest add use-toggle --cwd /path/to/project
```

```bash
npx uselab@latest add use-counter -c ./my-project
```

**Custom installation path:**

```bash
npx uselab@latest add use-counter --path src/custom/hooks
```

```bash
npx uselab@latest add use-toggle -p lib/hooks
```

**Install from URL:**

```bash
npx uselab@latest add https://example.com/hooks/use-custom.json
```

```bash
npx uselab@latest add https://cdn.example.com/hooks/use-utils.json
```

**Install from local file:**

```bash
npx uselab@latest add ./local-hooks/use-custom.json
```

```bash
npx uselab@latest add ../shared-hooks/use-utils.json
```

```bash
npx uselab@latest add /absolute/path/to/hook.json
```

**Combined options:**

```bash
npx uselab@latest add use-counter use-toggle --yes --path src/hooks
```

```bash
npx uselab@latest add use-network-status --overwrite --cwd ./project
```

#### Notes

- URLs and local paths must point to valid JSON files matching the registry hook format
- Multiple hooks are installed sequentially with progress tracking
- Running `add` without arguments displays usage instructions and examples
- The `--yes` flag automatically overwrites existing files without prompting

### `init` - Initialize Project

Initialize uselab configuration in your project by creating a `uselab.json` file.

#### Syntax

```bash
npx uselab@latest init [options]
```

#### Options

| Option             | Description                                                    | Default           |
| ------------------ | -------------------------------------------------------------- | ----------------- |
| `-c, --cwd <path>` | Working directory used to locate package.json and write config | Current directory |

#### Examples

**Initialize in current directory:**

```bash
npx uselab@latest init
```

**Initialize in specific directory:**

```bash
npx uselab@latest init --cwd /path/to/project
```

```bash
npx uselab@latest init -c ./my-project
```

#### Output

Creates a `uselab.json` file in your project root:

```json
{
  "hooks": "src/hooks"
}
```

### `view` - Preview Hook Details

View detailed information about hook(s) before installing, including code preview, dependencies, and file structure.

#### Syntax

```bash
npx uselab@latest view [options] [hooks...]
```

#### Arguments

- `[hooks...]` - Hook name(s) to view (space-separated, optional)
  - Shows usage help if omitted

  - Accepts kebab-case or camelCase format

#### Options

| Option              | Description                     | Default |
| ------------------- | ------------------------------- | ------- |
| `--format <format>` | Output format: `code` or `json` | `code`  |

#### Examples

**View a single hook:**

```bash
npx uselab@latest view use-network-status
```

```bash
npx uselab@latest view useToggle
```

```bash
npx uselab@latest view use-counter
```

**View multiple hooks:**

```bash
npx uselab@latest view use-counter use-toggle
```

```bash
npx uselab@latest view use-network-status useBattery useCounter
```

**View as JSON:**

```bash
npx uselab@latest view use-network-status --format json
```

```bash
npx uselab@latest view use-counter --format json
```

**View multiple hooks as JSON:**

```bash
npx uselab@latest view use-counter use-toggle --format json
```

#### Output Format

**Code format (default):**

- Human-readable format with syntax highlighting
- Shows code preview, dependencies, and file structure
- Displays installation instructions

**JSON format:**

- Complete hook data as JSON
- Includes all hook metadata and file contents
- Useful for programmatic access or integration

### `list` - List Available Hooks

List all available hooks from the UseKit registry with filtering and pagination support.

#### Syntax

```bash
npx uselab@latest list [options]
```

#### Options

| Option                  | Description                                                  | Default          |
| ----------------------- | ------------------------------------------------------------ | ---------------- |
| `-q, --query <query>`   | Filter hooks by query (searches in name, title, description) | None (shows all) |
| `-l, --limit <number>`  | Maximum number of results to show                            | `100`            |
| `-o, --offset <number>` | Number of results to skip for pagination                     | `0`              |
| `--skip-limit`          | Ignore pagination and show all results                       | `false`          |
| `--format <format>`     | Output format: `table` or `json`                             | `table`          |

#### Examples

**List all hooks:**

```bash
npx uselab@latest list
```

**Filter by query:**

```bash
npx uselab@latest list --query network
```

```bash
npx uselab@latest list -q "state management"
```

```bash
npx uselab@latest list --query counter
```

**Limit results:**

```bash
npx uselab@latest list --limit 20
```

```bash
npx uselab@latest list -l 50
```

**Pagination:**

```bash
npx uselab@latest list --offset 10
```

```bash
npx uselab@latest list --limit 20 --offset 10
```

```bash
npx uselab@latest list -l 10 -o 20
```

**Filter with pagination:**

```bash
npx uselab@latest list --query network --limit 10 --offset 10
```

```bash
npx uselab@latest list -q network -l 20 -o 0
```

**Show all results (ignore pagination):**

```bash
npx uselab@latest list --skip-limit
```

```bash
npx uselab@latest list --query network --skip-limit
```

```bash
npx uselab@latest list -q "state" --skip-limit
```

**JSON output:**

```bash
npx uselab@latest list --format json
```

```bash
npx uselab@latest list --format json --limit 10
```

```bash
npx uselab@latest list -q network --format json
```

**Combined options:**

```bash
npx uselab@latest list --query network --limit 20 --offset 10 --format json
```

```bash
npx uselab@latest list -q counter --skip-limit --format json
```

#### Output Format

**Table format (default):**

- Human-readable list with hook names, titles, and descriptions
- Shows pagination information
- Displays total count and current range

**JSON format:**

- Array of hook objects with only essential fields:

  ```json
  [
    {
      "name": "use-network-status",
      "title": "Use Network Status",
      "description": "Monitor network status..."
    }
  ]
  ```

### `search` - Search Hooks

Search for hooks in the registry with query filtering and pagination support. Similar to `list` but optimized for searching.

#### Syntax

```bash
npx uselab@latest search [options]
```

#### Options

| Option                  | Description                                                | Default          |
| ----------------------- | ---------------------------------------------------------- | ---------------- |
| `-q, --query <query>`   | Search query string (searches in name, title, description) | None (shows all) |
| `-l, --limit <number>`  | Maximum number of results to display                       | `100`            |
| `-o, --offset <number>` | Number of results to skip for pagination                   | `0`              |
| `--skip-limit`          | Ignore pagination and show all results                     | `false`          |

#### Examples

**Search all hooks:**

```bash
npx uselab@latest search
```

**Search with query:**

```bash
npx uselab@latest search network
```

```bash
npx uselab@latest search -q network
```

```bash
npx uselab@latest search --query "state management"
```

**Limit search results:**

```bash
npx uselab@latest search --limit 10
```

```bash
npx uselab@latest search -l 20
```

```bash
npx uselab@latest search network --limit 5
```

**Pagination:**

```bash
npx uselab@latest search --offset 10
```

```bash
npx uselab@latest search --limit 20 --offset 10
```

```bash
npx uselab@latest search network -l 10 -o 5
```

**Filter with pagination:**

```bash
npx uselab@latest search --query network --limit 20 --offset 10
```

```bash
npx uselab@latest search -q "state" -l 15 -o 0
```

**Show all results (ignore pagination):**

```bash
npx uselab@latest search --skip-limit
```

```bash
npx uselab@latest search network --skip-limit
```

```bash
npx uselab@latest search -q "state management" --skip-limit
```

**Combined options:**

```bash
npx uselab@latest search --query network --limit 20 --offset 10
```

```bash
npx uselab@latest search -q counter --skip-limit
```

#### Differences from `list`

- `search` is optimized for finding hooks with queries
- `list` supports JSON output format, `search` does not
- Both support the same filtering and pagination options
- `search` uses fuzzy matching for better discovery

## Configuration

The CLI creates a `uselab.json` file in your project root when you run `init`:

```json
{
  "hooks": "src/hooks"
}
```

### Hooks Directory

By default, hooks are installed to:

- `src/hooks/` if a `src` directory exists
- `hooks/` in the project root otherwise

You can customize this in the `uselab.json` configuration file, or use the `--path` option when installing hooks.

### Configuration Options

| Option  | Description                      | Default                 |
| ------- | -------------------------------- | ----------------------- |
| `hooks` | Relative path to hooks directory | `src/hooks` or `hooks/` |

## Hook Sources

The `add` command supports three types of hook sources:

### 1. Registry Hooks

Install hooks from the official UseKit registry:

```bash
npx uselab@latest add use-network-status
```

```bash
npx uselab@latest add useToggle
```

### 2. URL Hooks

Install hooks from a remote URL:

```bash
npx uselab@latest add https://example.com/hooks/use-custom.json
```

```bash
npx uselab@latest add https://cdn.example.com/hooks/use-utils.json
```

**Requirements:**

- URL must point to a valid JSON file
- JSON must match the registry hook format
- File must be accessible via HTTP/HTTPS

### 3. Local File Hooks

Install hooks from a local file path:

```bash
npx uselab@latest add ./local-hooks/use-custom.json
```

```bash
npx uselab@latest add ../shared-hooks/use-utils.json
```

```bash
npx uselab@latest add /absolute/path/to/hook.json
```

**Requirements:**

- Path must point to a valid JSON file
- JSON must match the registry hook format
- File must be readable
